# 주간 해외매크로 보고서 자동화 — 작업 이력 (2026-05-12)

## 배경 및 목표

글로벌 매크로 경제 데이터를 매주 JSON 파일로 정리하면 **자동으로 보고서를 생성**하는 시스템을 구축한다.
출력 형식은 두 가지: ① 인쇄용 `.docx` ② 인터랙티브 웹 대시보드 `.html`

---

## 구현 단계별 작업 내역

### 1단계 — docx 보고서 생성기 (`generate_report.py`)

- Python + `python-docx` 라이브러리로 Malgun Gothic 폰트 A4 보고서 생성
- 5개 섹션 구성:
  1. 주간 핵심 요약
  2. 글로벌 금융시장 동향 (증시·채권·원자재·FX·경제지표·연준)
  3. 핵심 이슈 뉴스 (사실/의견 구분)
  4. 다음 주 일정 (중요도 ★ 표시)
  5. 시사점 및 전망
- CLI: `python generate_report.py` / `python generate_report.py 20260519` / `python generate_report.py data/file.json`
- 출력: `archive/YYYY/Weekly foreign-macro_YYYYMMDD.docx`

### 2단계 — 프로젝트 관리 문서 정비

| 파일 | 내용 |
|------|------|
| `README.md` | 환경 설정·실행 방법·파일 구조·데이터 소스 |
| `CHECKLIST.md` | 10단계 주간 데이터 수집 체크리스트 (소스 URL 포함) |
| `Weekly_Foreign_Macro_Template.md` | 5섹션 + 선택 4섹션 빈 양식 |
| `requirements.txt` | `python-docx==1.2.0`, `lxml==6.1.0` |
| `.gitignore` | `.venv/`, `__pycache__/`, 임시파일 등 |

### 3단계 — 데이터/코드 분리

- 모든 주차 데이터를 `data/data_YYYYMMDD.json`으로 분리
- 구조: `meta` → `summary` → `markets` (6 소섹션) → `news` → `next_week` → `outlook`
- 각 항목에 `source` URL 필드 추가 (원본 출처 링크)
- `data/data_20260512.json`: 2026-05-05 ~ 05-11 보고 기준, 실제 데이터 입력 완료

### 4단계 — 웹 대시보드 생성기 (`generate_html.py`)

**두 가지 HTML 파일 생성:**

| 출력 | 용도 |
|------|------|
| `archive/index.html` | 인터랙티브 대시보드 (날짜 선택·탭·차트) |
| `archive/YYYY/Weekly foreign-macro_YYYYMMDD.html` | 인쇄용 단일 보고서 |

**대시보드 주요 기능:**

- **날짜 선택기**: 상단 드롭다운 → `REPORTS` JS 객체에서 해당 날짜 데이터 로드
- **시장 탭**: 증시 / 채권금리 / 원자재 / FX / 경제지표 / 연준
- **KPI 칩**: 핵심 수치 한눈에 표시
- **뉴스 아코디언**: 이슈 클릭 → 배경/경과/영향/의견 펼침
- **과거 데이터 차트**: `data/historical_data.json` 기반 Chart.js 라인 차트 (약 3년치)
- **출처 링크**: 각 데이터 옆 소스 URL 표시

**기술 스택:**

- Python f-string으로 HTML 직접 생성 (추가 패키지 없음)
- CSS·JS 전체 인라인 임베드 (file:// 로컬 실행 가능)
- Chart.js 4.4.3 CDN (차트 렌더링 시 인터넷 필요)
- 모든 보고서 데이터를 `REPORTS` JS 상수로 임베드 → 서버 불필요

## 주간 워크플로우 (매주 반복)

```powershell

copy data\data_20260512.json data\data_20260519.json

# 2. JSON 파일에서 데이터 수정
#    meta.date, meta.period, summary, markets, news, next_week, outlook

# 3. 보고서 생성
.venv\Scripts\python.exe generate_report.py   # docx

# archive\index.html 더블클릭
```

## 해결한 기술 이슈

| 문제 | 해결책 |
|------|--------|
| uv Python externally-managed 오류 | `uv venv .venv` 생성 후 해당 Python으로 pip 실행 |
| `print()` UnicodeEncodeError (한글) | `sys.stdout.buffer.write(...encode("utf-8"))` 사용 |
| file:// 프로토콜에서 fetch() 불가 | 모든 데이터를 JS 상수로 HTML에 인라인 임베드 |
| 날짜별 보고서 선택 | `REPORTS` 객체에 전체 날짜 데이터 저장, `switchReport()` 함수로 동적 렌더링 |
